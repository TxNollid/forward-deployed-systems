# Phase 5: Enablement

Weeks 6-10. Goal: transfer knowledge so the client can own, operate, and evolve the solution without you.

## Objectives

1. Train end users on the new workflow
2. Train technical staff on system operations
3. Document everything needed for independent operation
4. Build internal champions who will sustain adoption

## Activities

### User Training
- Hands-on workshops with actual workflows, not slides
- Create quick-reference guides for daily use
- Address the "what do I do when it's wrong?" question explicitly
- Identify power users who can train others

### Technical Training
- System architecture walkthrough with the client's engineering team
- Runbook review — monitoring, alerting, common issues, escalation
- Code walkthrough if client will maintain/extend
- Infrastructure management — scaling, updates, costs

### Documentation
- User guide for end users
- Technical operations guide for IT/engineering
- Architecture decision records (why, not just what)
- Known limitations and workaround guide

### Champion Building
- Identify 2-3 internal advocates at different levels
- Equip them with success metrics to share upward
- Connect them to the broader AI/ML community if appropriate
- Set them up to lead the next phase of AI adoption internally

## Anti-Patterns

- Training once and leaving — plan for reinforcement
- Only training managers — train the people who actually use it
- Documentation dump — nobody reads 50 pages. Make it scannable
- Creating dependency on the FDE — if they can't run it without you, enablement failed

## Outputs

- Training materials (guides, reference cards, recorded walkthroughs)
- Technical documentation and runbooks
- Identified internal champions with action plans
- Support escalation path defined
