### How It Works Section

**Eyebrow:** HOW IT WORKS

**Headline:**
Input → Action → Output,
the AI kernel of processes.

**SDLC phase labels (num / name / short input → output):**
1. Scoping — Client request, domain context → Signed SOW, stakeholder map
2. Discovery — SOW, stakeholder map → Process maps, data inventory
3. Requirements — Findings, KPIs → Use cases, acceptance criteria
4. Design — Requirements, tech landscape → Architecture, integration spec
5. Build — Architecture, data access → Code, model artifacts
6. Test — Code, acceptance criteria → Validation report
7. Deploy — Validated code, infra → Production system, runbook
8. Enable — Production system → Trained users, documentation
9. Operate — System, monitoring → KPI tracking, optimizations
10. Handoff — Metrics, documentation → Outcomes report, roadmap
