# Phase 01: Scoping

Define the engagement boundaries, align with the client, and set up for success.

---

## Inputs

| Input | Source | Notes |
|-------|--------|-------|
| Client request / RFP | Client | The initial ask — may be vague |
| Domain overview | `domains/{industry}/overview.md` | Industry landscape, common pain points |
| Domain roles | `domains/{industry}/roles.md` | Expected org structure |
| Domain AI opportunities | `domains/{industry}/ai-opportunities.md` | Known high-value use cases |

## Actions

### 1. Qualify the Engagement
- Review the client request against domain knowledge
- Assess: Is this a real problem? Is AI the right solution? Is the client ready?
- Identify which domain AI opportunities align with the client's stated need

### 2. Map Initial Stakeholders
- Identify executive sponsor, day-to-day POC, technical gate (CIO/IT)
- Reference `domains/{industry}/roles.md` for expected titles and motivations
- Use `skills/stakeholder-mapping.md` for methodology

### 3. Define Scope
- What's in, what's out — be explicit
- Tie scope to specific processes from `domains/{industry}/processes.md`
- Set expectations on timeline, resources, and client responsibilities

### 4. Draft SOW
- Use `templates/sow.md`
- Include objectives tied to domain KPIs (`domains/{industry}/kpis.md`)
- Define acceptance criteria (what "done" looks like)

### 5. Plan the Engagement
- Map phases to calendar
- Identify access and resource needs
- Schedule kickoff meeting

## Outputs

| Output | Template/Format | Consumed By |
|--------|----------------|-------------|
| Signed SOW | `templates/sow.md` | All phases — defines the contract |
| Stakeholder map (initial) | `skills/stakeholder-mapping.md` | 02 Discovery |
| Engagement plan | Timeline + milestones | 02 Discovery |
| Access request list | Checklist | 02 Discovery |

## Quality Gate

- [ ] SOW signed by both parties
- [ ] Executive sponsor identified and engaged
- [ ] Day-to-day POC confirmed
- [ ] Access requests submitted
- [ ] Kickoff meeting scheduled
